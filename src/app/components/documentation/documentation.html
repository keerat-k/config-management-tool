Overview

The Configuration Management Tool replaces Excel-based workflow configuration files with a structured, database-backed interface. Each workflow defines a sequence of data processing steps that are executed by the Data Profiling Notebook in Azure Databricks to prepare datasets for Power BI dashboards.

Workflow List

The main page displays all configured workflows. Each row shows the Workflow ID, Category, Name, Business Unit, Facility, Payload, and the number of steps.

  • Single-click a row to select it.
  • Double-click a row to open the workflow and view its steps.
  • Click Documentation in the toolbar to view this page.

Workflow Detail

The detail page shows all steps for a workflow in execution order. Steps are the individual data operations applied to datasets in sequence.

  • Click a row to select it and enable the action buttons.
  • Drag a row to reorder steps.
  • Add Step opens a modal to create a new step.
  • Edit Row opens the modal pre-filled with the selected step's data.
  • Duplicate Row creates a copy of the selected step at the end of the list.
  • Delete Row permanently removes the selected step.

View Additional Info

The View Additional Info dropdown in the toolbar allows you to show optional columns that are hidden by default to reduce visual clutter.

  • Dataset 2 — shown in yellow. The secondary input dataset for operations like Join.
  • Output Parameters — shown in green. The name of the output dataset produced by the step.

Data Operations

Each step has a Data Operation that defines what transformation is applied to the dataset.

Ingest Loads raw data from a source system into the pipeline. Specify the source path or connection in Operation Specific Info.
AddCol Adds a new column to the dataset. Specify the column name and default value or formula in Operation Specific Info.
RenameCol Renames an existing column. Specify old_name and new_name in Operation Specific Info.
Join Joins two datasets. Specify the join key and join type (LEFT, INNER, etc.) in Operation Specific Info. Dataset 2 is required.

Fields Reference

Field Description
Workflow ID Unique identifier for the workflow (e.g. WF-001).
Workflow Category The type of workflow: Ingestion, Transformation, or Reporting.
Business Unit The Chevron business unit that owns this workflow.
Facility The physical facility or location associated with this workflow.
Payload The name of the data payload or package this workflow processes.
Step ID Auto-generated identifier for the step (e.g. S-001).
Dataset The dataset identifier this step operates on (e.g. DS-RAW-001).
Dataset 1 Name The primary input dataset name for this step.
Dataset 2 Name The secondary input dataset, used for Join operations.
Output Parameters The name of the output dataset produced by this step.
Operation Specific Info Additional configuration specific to the data operation type.
